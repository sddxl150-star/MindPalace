🤖 Claude Code 技术架构解析

基于 npm Source Map 泄露事件 · 深度剖析 AI 编程代理的工程实践
2026年3月31日 · 512,000行TypeScript代码全解析

⏰ 事件时间线

2026年3月31日，安全研究员 Chaofan Shou 发现 @anthropic-ai/claude-code v2.1.88 npm 包中包含了一个 59.8 MB 的 JavaScript Source Map 文件。 Source Map 是调试工具，用于将压缩后的生产代码还原为可读的源代码——它们永远不应该公开发布。

这次泄露源于 Bun 打包器默认生成 Source Map（一个 已知 Bug）， Anthropic 未能在发布配置中排除 *.map 文件。

结果：约 512,000 行 TypeScript，跨越 1,906 个源文件，全部公开可读。 Shou 在 X 上发布了截图，1,600万人查看了这条帖子。

8大核心创新

Claude Code 的工程智慧

1. 流式工具执行 Streaming

模型仍在生成 token，但工具已经开始运行。当 LLM 发出 tool_use_stop 事件时，执行器立即触发——无需等待完整响应。

tool_use_stop 事件触发 → 执行器立即响应 → 无需等待完整响应 → 最大化利用等待时间

2. 并发安全调度 Concurrency

每个工具声明 isConcurrencySafe。只读工具（Read、Glob、Grep）并行运行。变更工具（Bash、Write、Edit）串行运行。

Bash 错误取消同级工具；其他工具失败相互独立，不影响并行任务

3. 分层系统提示 Prompt

静态指令放在第一个带 cache_control:ephemeral 的块中。动态上下文（cwd、git status、CLAUDE.md、memory）放在后续块中。昂贵的静态块被 API 缓存。

第一块（静态）→ 缓存复用 → 节省 API 调用成本

4. 上下文压缩 Compression

Token预算感知的渐进式压缩：截断 → LLM 摘要 → 断路器（3次失败）。最近消息保持原文；较早消息被压缩。

截断 → LLM摘要 → 断路器(3次失败) → 平衡精度与成本

5. 多Provider架构 Multi-Provider

统一的 Provider 接口。Anthropic（原生流式+提示缓存）、OpenAI，以及任何 OpenAI 兼容端（Ollama、Together、Groq、LM Studio、vLLM）。

One Provider Interface → Anthropic / OpenAI / 兼容端 → 灵活切换

6. 权限系统 Permissions

三种模式：auto（智能按工具分析）、ask（确认所有写操作）、bypass。Bash 命令通过安全/危险命令列表+模式匹配。MCP 工具默认 ask。

auto / ask / bypass → 智能分析 → 危险操作拦截

7. 子Agent Sub-Agents

LLM 可以生成子Agent处理子任务。隔离上下文（全新对话）、链接中止（父取消→子取消）、深度限制（最大3层）、共享 provider 和权限策略。

隔离上下文 → 链接中止 → 深度限制(3层) → 并行处理复杂任务

8. 延迟工具加载 Deferred

核心工具立即发送。MCP 和工作流工具延迟加载——通过 ToolSearch 发现。注册大量工具时节省约 40% 的提示 token。

核心工具 → 立即加载 | MCP/工作流 → 延迟加载 → 节省40% token

1

接收用户输入

循环迭代

五层上下文压缩

上下文永不溢出的秘密

Layer 1: 工具结果预算 Tool Result Budget

单个工具结果的最大 token 数限制。超出部分截断并标记。

Layer 2: 历史裁剪 History Snip

渐进式裁剪对话历史，保留最近消息的完整性。

Layer 3: 微观压缩 Microcompact

LLM 摘要最近对话，保留关键信息，压缩冗余内容。

Layer 4: 上下文崩溃 Context Collapse

实验性功能，深度压缩历史消息池。

Layer 5: 自动压缩 Autocompact

断路器机制：连续3次压缩失败时触发紧急压缩。

八层安全架构

企业级安全保障

权限模式 Permissions

auto: 智能分析每个工具
ask: 确认所有写操作
bypass: 完全信任模式

命令白名单/黑名单 Command Lists

安全命令列表 + 危险命令列表。拒绝列表始终优先于允许列表。

子命令分析 Sub-command

复合 Bash 命令按子命令逐一评估，确保每个子命令都安全。

MCP 工具默认ask MCP Security

Model Context Protocol 工具默认需要用户确认，防止未知操作。

错误隔离 Error Isolation

Bash 错误取消同级工具；其他工具失败相互独立。

特性标志控制 Feature Flags

GrowthBook 驱动的服务端特性控制，可远程禁用危险功能。
